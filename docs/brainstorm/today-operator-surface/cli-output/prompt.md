# Task

A cluster of four related kanban tasks shipping as one release for the Open Second Brain project (Bun + TypeScript CLI `o2b`, Markdown vault, brain memory subsystem, MCP server).

## Task A (t_5f65c992) - Sectioned read-only today dashboard

A read-only, multi-section "today" overview generated by the brain - distinct from the existing morning-brief by being domain-sectioned (due/overdue obligations, open loops, recent activity, totals) where each section is an independent live query over vault notes, never stored content. The dashboard re-derives every section on demand so it is maintenance-free.

## Task B (t_4adb0b8b) - Typed, age-labeled recent-activity timeline

At session start, format recalled/loaded memories as a scannable chronological "Recent Activity" timeline: each entry prefixed with a type tag and a relative-age label ("2h ago", "1d ago"). NOTE: the section-grouped typed bullets with age labels ALREADY shipped (morning-brief.ts timelineBullet + relativeAge). The remaining gap is a genuinely chronologically MERGED timeline across event kinds, rendered from the existing TimelineIndex (buildTimelineIndex sorts all BrainLogEventKind events chronologically already, but nothing renders it as a human timeline or wires it into a session-start/dashboard surface).

## Task C (t_97373418) - Open-loops carry-forward from prose markers

Scan prior-period daily notes / narrative prose for lightweight inline open-loop markers and surface those unfinished items in today's brief until explicitly closed. Lighter-weight than a recurring obligation - captures intentions noted in passing. Closing convention needs definition. Must be language-agnostic: the marker is a structural token, not a natural-language word list.

## Task D (t_d7be2a0c) - Prose markers into logged frontmatter mutations

Parse imperative completion/update markers the operator jots into daily prose, resolve them to the referenced notes, and apply property mutations - logging each change. Must be deterministic and logged (one log event per applied mutation); ambiguous note references surface as a conflict rather than guess.
